# Next.js란?

Next.js는 React 기반의 **프레임워크**입니다. React가 단순히 UI를 구축하기 위한 라이브러리라면, Next.js는 React를 기반으로 웹 애플리케이션을 개발하기 위한 완전한 솔루션을 제공합니다.

## 왜 Next.js를 사용할까?

### 1. 서버 사이드 렌더링 (SSR) 지원

React는 기본적으로 클라이언트 사이드 렌더링(CSR)만 지원합니다. 하지만 Next.js는 서버에서 페이지를 렌더링할 수 있어, 초기 로딩 속도가 빠르고 SEO에 유리합니다.

### 2. 파일 기반 라우팅

별도의 라우터 설정 없이 `pages` 폴더에 파일을 만들기만 하면 자동으로 라우트가 생성됩니다. 예를 들어 `pages/about.js`를 만들면 `/about` 경로가 자동으로 생성됩니다.

### 3. 코드 스플리팅 자동화

각 페이지별로 필요한 코드만 로드되도록 자동으로 최적화됩니다. 이는 번들 크기를 줄이고 성능을 향상시킵니다.

### 4. 이미지 최적화

`next/image` 컴포넌트를 통해 이미지를 자동으로 최적화하고, lazy loading을 지원합니다.

### 5. API Routes

백엔드 API를 별도로 구축하지 않고도 `pages/api` 폴더에 파일을 만들면 API 엔드포인트를 만들 수 있습니다.

### 6. 빌드 최적화

프로덕션 빌드 시 자동으로 코드를 최적화하고, 정적 페이지는 미리 생성(Static Site Generation)할 수 있습니다.

## 라이브러리 vs 프레임워크

이 둘의 차이를 이해하는 것이 중요합니다.

### 라이브러리 (Library)

- **제어권이 개발자에게 있음**
- 필요한 기능만 선택적으로 사용
- 코드의 흐름을 개발자가 결정
- 예: React, jQuery, Lodash

**예시:**

```javascript
// React를 사용할 때 - 내가 언제, 어떻게 사용할지 결정
import React from "react";

function App() {
  return <div>Hello</div>;
}
```

### 프레임워크 (Framework)

- **제어권이 프레임워크에게 있음**
- 프레임워크가 전체 구조를 제공
- 개발자는 프레임워크가 정한 규칙에 따라 코드를 작성
- 예: Next.js, Angular, Django

**예시:**

```javascript
// Next.js를 사용할 때 - Next.js가 파일 구조와 규칙을 정해줌
// pages/index.js - 자동으로 '/' 경로가 됨
export default function Home() {
  return <div>Hello</div>;
}
```

### 핵심 차이점

**라이브러리:**

- 유연하지만 모든 것을 직접 구성해야 함
- 작은 단위의 기능 제공

**프레임워크:**

- 구조가 정해져 있어 빠르게 시작 가능
- 전체적인 아키텍처 제공

### React는 라이브러리, Next.js는 프레임워크

React 자체는 UI를 만드는 라이브러리입니다. 하지만 Next.js는 React를 기반으로 웹 애플리케이션을 만들기 위한 프레임워크입니다.

- **React만 사용**: 라우팅, SSR, 빌드 설정 등을 모두 직접 해야 함
- **Next.js 사용**: 이런 것들이 이미 준비되어 있어서 바로 개발에 집중할 수 있음

## 결론

Next.js는 React로 현대적인 웹 애플리케이션을 만들 때 필요한 많은 기능들을 미리 제공해주는 프레임워크입니다. 특히 SEO가 중요하거나, 빠른 초기 로딩이 필요한 프로젝트에서 매우 유용합니다.
