# React 상태 관리 라이브러리 총정리 (2025 ver.) 📚

리액트 내장 기능(`Context API`)만으로는 성능 최적화나 복잡한 로직 처리에 한계가 있어 외부 라이브러리를 사용합니다.
크게 **Flux(중앙 집중형)**, **Atomic(분산형)**, **Proxy(관찰형)**, **Server State(서버 동기화)**로 나뉩니다.

---

## 1. Flux / 중앙 집중형 (Store 방식)

> **"하나의 거대한 중앙 금고(Store)에 모든 데이터를 몰아넣고 관리한다."**
> 가장 전통적이고 검증된 방식입니다.

### ① Redux Toolkit (RTK) 🦍

- **지위:** **업계 표준 (Standard)**. 가장 많이 쓰이고, 취업 시장에서 가장 많이 요구함.
- **특징:** 예전 Redux의 복잡한 설정(Boilerplate)을 획기적으로 줄여줌.
- **장점:** 강력한 디버깅 도구(DevTools), 거대한 커뮤니티, 예측 가능한 데이터 흐름.
- **단점:** 여전히 코드가 좀 긺. 작은 프로젝트에는 과함.

### ② Zustand (주스탄드) 🐻

- **지위:** **요즘 대세 (Rising Star)**.
- **특징:** "Redux를 최대한 쉽고 가볍게 만들자"는 철학. 독일어로 '상태'라는 뜻.
- **장점:**
  - **엄청나게 쉬움.** (설정 코드가 거의 없음)
  - **매우 가벼움.** (번들 사이즈 작음)
  - `Context Provider`로 감쌀 필요가 없음.
- **추천:** **개인 프로젝트나 스타트업의 신규 프로젝트 1순위.**

---

## 2. Atomic / 분산형 (Atom 방식)

> **"데이터를 잘게 쪼개서(Atom) 필요한 곳에서만 조립해 쓴다."**
> 리액트의 `useState` 느낌을 그대로 살리면서 전역으로 확장한 방식입니다.

### ① Recoil (리코일) ⚛️

- **제작:** Meta(Facebook) 팀이 만듦. "리액트스러운" 상태 관리를 지향.
- **특징:** `atom`(상태)과 `selector`(계산된 상태) 개념 사용.
- **장점:** 리액트 문법과 가장 비슷해서 배우기 쉬움.
- **단점:** **업데이트가 멈춤.** (Meta에서 유지보수를 거의 안 해서 Jotai로 넘어가는 추세)

### ② Jotai (조타이) 👻

- **지위:** **Recoil의 강력한 대안.** (일본어로 '상태'라는 뜻)
- **특징:** Recoil의 장점은 가져오고, 단점(무거운 크기, 키 관리)은 없앰.
- **장점:** `useState`처럼 쓰면 됨. 아주 가볍고 유연함.
- **추천:** 서로 얽히고설킨 상태가 많은 앱(예: 엑셀, 그래픽 툴 등).

---

## 3. Proxy / 관찰형 (Mutable 방식)

> **"데이터를 그냥 수정하면, 라이브러리가 알아서 감지하고 화면을 고쳐준다."**
> 불변성(`...state`) 신경 안 쓰고 편하게 코딩하고 싶을 때 씁니다.

### ① MobX (몹엑스) 🧐

- **특징:** 객체 지향 프로그래밍(OOP) 느낌이 강함.
- **장점:** 코드가 직관적임. (`store.count++` 하면 알아서 렌더링 됨)
- **단점:** "마법"이 너무 많아서 디버깅이 어려울 수 있음. 리액트의 철학과는 조금 다름.

### ② Valtio 🖌️

- **특징:** MobX를 더 단순하게 만든 버전. Proxy 기반.

---

## 4. Server State / 서버 데이터 전용

> **"서버에서 가져온 데이터(API)는 이걸로 관리해라."**
> 클라이언트 상태(테마, 모달)와 서버 상태(DB 데이터)를 분리하는 것이 현대 개발의 트렌드입니다.

### ① TanStack Query (구 React Query) 🔴

- **지위:** **사실상 필수 (De facto standard).**
- **역할:** API 호출, 캐싱(저장), 로딩 처리, 에러 처리, 자동 새로고침 등을 다 해줌.
- **사용법:** 보통 **[Zustand(클라이언트) + TanStack Query(서버)]** 조합으로 많이 씀.

---

## ⚡️ 요약: 뭐 써야 해요? (추천 가이드)

1. **취업 준비 / 대규모 프로젝트 유지보수** 👉 **Redux Toolkit** (무조건 알아야 함)
2. **신규 프로젝트 / 빠르고 간편한 개발** 👉 **Zustand** (가장 추천 👍)
3. **복잡한 계산이 많거나 리액트 문법이 좋다면** 👉 **Jotai**
4. **API 통신이 많은 앱** 👉 **TanStack Query** (이건 상태 관리랑 별개로 무조건 쓰세요)
