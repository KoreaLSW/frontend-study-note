# SPA에서 정적 Meta 태그 입력의 한계와 동적 SEO

## 1. "HTML 파일에 그냥 Meta 태그를 적으면 안 되나요?"

결론: **페이지가 하나뿐인 사이트라면 OK, 하지만 페이지가 여러 개인 서비스(쇼핑몰 등)라면 불가능.**

### ❌ 안 되는 이유: "파일이 하나뿐이라서"

SPA는 물리적인 HTML 파일이 **`index.html` 단 하나**입니다.

- **상황**: `index.html`에 `<title>신발 가게</title>`라고 적어서 서버에 업로드.
- **문제**:
  - 사용자가 **나이키 운동화 상세 페이지**에 접속해도 제목은 여전히 **"신발 가게"**.
  - 사용자가 **아디다스 모자 상세 페이지**에 접속해도 제목은 여전히 **"신발 가게"**.
- **원인**: 사용자가 어떤 주소로 접속하든 서버는 항상 **똑같은 `index.html`**만 보내주기 때문.

---

## 2. SNS 공유 시 발생하는 문제 (카카오톡 예시)

친구에게 **"나이키 신발"** 페이지 링크를 카카오톡으로 공유했을 때:

- **기대**: 나이키 신발 사진과 "나이키 에어맥스"라는 제목.
- **현실**: 가게 기본 로고와 "신발 가게"라는 기본 제목만 뜸.
- **이유**: 카카오톡 봇은 링크를 받으면 `index.html`의 소스 코드만 긁어가는데, 거기엔 우리가 처음에 적어둔 **기본 정보(신발 가게)**밖에 없기 때문.

---

## 3. JS로 바꾸면 되지 않나요? (Dynamic Meta Tag)

브라우저에서 자바스크립트로 `document.title = "나이키 신발"`처럼 바꿀 수는 있습니다.

- **사람(사용자)**: JS가 실행되면서 제목이 바뀌는 것을 볼 수 있음. (성공)
- **검색 봇/SNS 봇**: JS를 실행하지 않거나 기다리지 않고, **처음 받은 HTML 코드(신발 가게)**만 보고 떠남. (실패)

---

## 4. 해결책과 결론

| 상황                        | index.html 직접 입력 가능? | 이유                                         |
| :-------------------------- | :------------------------- | :------------------------------------------- |
| **랜딩 페이지, 포트폴리오** | **⭕️ 가능**               | 페이지가 한 장뿐이고 내용이 고정이라서.      |
| **쇼핑몰, 블로그, 뉴스**    | **❌ 불가능**              | 페이지마다 제목/설명/이미지가 달라져야 해서. |

### ✅ 해결 방법: SSR (Server Side Rendering)

**Next.js** 같은 기술을 사용하면,

1. 사용자가 `/product/nike`로 접속 요청.
2. **서버에서** 즉시 `<title>나이키 신발</title>`로 내용을 갈아 끼운 HTML을 생성.
3. 봇과 사용자에게 전달하므로 **SEO와 SNS 공유 모두 완벽 해결**.
