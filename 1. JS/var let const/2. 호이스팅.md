# 호이스팅(Hoisting)의 차이

## 호이스팅이란?

**호이스팅(Hoisting)**은 JavaScript 엔진이 코드를 실행하기 전에 변수 선언과 함수 선언을 해당 스코프의 최상단으로 끌어올리는 동작을 의미합니다.

### 중요한 점

- 호이스팅은 **선언(declaration)**만 끌어올리고, **할당(assignment)**은 끌어올리지 않습니다.
- 코드가 실제로 위로 이동하는 것이 아니라, JavaScript 엔진이 내부적으로 처리하는 방식입니다.
- `var`, `let`, `const` 모두 호이스팅되지만, 동작 방식이 다릅니다.

### 호이스팅이 발생하는 이유

1. JavaScript는 코드를 실행하기 전에 전체 코드를 스캔하여 변수와 함수 선언을 찾습니다.
2. 이를 메모리에 저장하고 스코프를 설정합니다.
3. 이 과정에서 선언이 스코프의 최상단으로 끌어올려진 것처럼 동작합니다.

## var - 호이스팅되며 undefined로 초기화

```javascript
console.log(a); // undefined (에러가 나지 않음!)
var a = 5;
```

`var`로 선언한 변수는 호이스팅되어 선언부가 최상단으로 끌어올려지고, `undefined`로 초기화됩니다.

## let, const - 호이스팅되지만 초기화되지 않음 (TDZ)

```javascript
console.log(b); // ReferenceError: Cannot access 'b' before initialization
let b = 5;

console.log(c); // ReferenceError: Cannot access 'c' before initialization
const c = 10;
```

`let`과 `const`도 호이스팅되지만, 초기화되기 전까지는 접근할 수 없는 **Temporal Dead Zone(TDZ)**에 있습니다.
