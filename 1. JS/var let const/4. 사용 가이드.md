# var, let, const 사용 가이드

## 언제 무엇을 사용해야 할까?

### const 사용 권장

- 기본적으로 모든 변수는 `const`로 선언
- 값이 변경되지 않는 경우

### let 사용

- 변수 값이 변경되어야 하는 경우
- 반복문의 카운터 등

### var 사용

- **현대 JavaScript에서는 사용하지 않는 것을 권장**
- 레거시 코드나 특별한 경우에만 사용

## 요약 표

| 특징             | var                | let         | const       |
| ---------------- | ------------------ | ----------- | ----------- |
| 스코프           | 함수 스코프        | 블록 스코프 | 블록 스코프 |
| 호이스팅         | undefined로 초기화 | TDZ         | TDZ         |
| 재선언           | 가능               | 불가능      | 불가능      |
| 재할당           | 가능               | 가능        | 불가능      |
| 초기화 없이 선언 | 가능               | 가능        | 불가능      |
| 전역 객체 속성   | 됨                 | 안 됨       | 안 됨       |

## 실전 팁

1. **기본적으로 `const` 사용**: 값이 변경되지 않는다면 `const`를 사용하세요.
2. **값 변경이 필요할 때만 `let` 사용**: 변수 값이 변경되어야 하는 경우에만 `let`을 사용하세요.
3. **`var`는 피하세요**: ES6 이후로는 `var` 사용을 피하는 것이 좋습니다.
4. **객체/배열은 `const`로 선언**: 객체나 배열 자체를 재할당하지 않는다면 `const`를 사용해도 내부 값 변경은 가능합니다.

```javascript
// 좋은 예시
const user = { name: "철수" };
user.name = "영희"; // 가능

let counter = 0;
counter++; // 값 변경이 필요하므로 let 사용

// 나쁜 예시
var oldStyle = "사용하지 마세요";
```
