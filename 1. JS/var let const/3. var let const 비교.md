# var, let, const 비교

## 1. 재선언(Redecaration) 가능 여부

### var - 재선언 가능

```javascript
var name = "철수";
var name = "영희"; // 에러 없이 재선언 가능
console.log(name); // "영희"
```

### let, const - 재선언 불가능

```javascript
let age = 20;
let age = 25; // SyntaxError: Identifier 'age' has already been declared

const city = "서울";
const city = "부산"; // SyntaxError: Identifier 'city' has already been declared
```

## 2. 재할당(Reassignment) 가능 여부

### var, let - 재할당 가능

```javascript
var score = 100;
score = 90; // 가능

let count = 5;
count = 10; // 가능
```

### const - 재할당 불가능

```javascript
const PI = 3.14;
PI = 3.14159; // TypeError: Assignment to constant variable
```

**주의**: `const`는 재할당이 불가능하지만, 객체나 배열의 내부 값은 변경할 수 있습니다.

```javascript
const person = { name: "철수" };
person.name = "영희"; // 가능! (객체 내부 변경)
person = {}; // TypeError (재할당은 불가능)

const arr = [1, 2, 3];
arr.push(4); // 가능! (배열 내부 변경)
arr = []; // TypeError (재할당은 불가능)
```

## 3. 초기화 없이 선언 가능 여부

### var, let - 초기화 없이 선언 가능

```javascript
var x;
let y;
console.log(x); // undefined
console.log(y); // undefined
```

### const - 선언과 동시에 초기화 필수

```javascript
const z; // SyntaxError: Missing initializer in const declaration
const z = 10; // 올바른 사용법
```

## 4. 전역 객체 속성으로 추가되는지 여부

### var - 전역 객체(window/global)의 속성이 됨

```javascript
var globalVar = "전역";
console.log(window.globalVar); // "전역" (브라우저 환경)
```

### let, const - 전역 객체의 속성이 되지 않음

```javascript
let globalLet = "전역";
const globalConst = "전역";
console.log(window.globalLet); // undefined
console.log(window.globalConst); // undefined
```

## 5. 반복문에서의 동작 차이

### var를 사용한 경우 (문제 발생)

```javascript
for (var i = 0; i < 3; i++) {
  setTimeout(() => {
    console.log(i); // 3, 3, 3 (의도와 다름!)
  }, 100);
}
```

### let을 사용한 경우 (올바른 동작)

```javascript
for (let i = 0; i < 3; i++) {
  setTimeout(() => {
    console.log(i); // 0, 1, 2 (의도한 대로!)
  }, 100);
}
```

`var`는 함수 스코프이기 때문에 반복문이 끝난 후의 `i` 값(3)을 참조하지만, `let`은 각 반복마다 새로운 블록 스코프를 생성하여 올바르게 동작합니다.
